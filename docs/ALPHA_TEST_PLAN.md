# Alpha Test Plan

Use this plan for `v0.1.0-alpha.2-candidate` external testing.

## Alpha Validation Bundle Workflow

For each real city export, generate a validation bundle first:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\generate-alpha-validation-bundle.ps1 -InputJson D:\CS2MetroDiagram\metro-export.json -CaseName <city-or-case-name>
```

Review and attach the generated bundle under:

```text
artifacts\alpha-validation\<timestamp>-<caseName>
```

The bundle contains the geographic alpha baseline, schematic-lite comparison, schematic-v2 comparison, visual continuity report, schematic-v2 diagnostics, notes, and a filled feedback template. Prefer this bundle over scattered screenshots or loose SVG files.

- Shared corridor and express stripe should not block alpha testing.
- Do not treat shared corridor / express stripe output as the default recommendation.
- Phase 5A.9 Route Run Stitcher is not started for alpha.2 candidate because the accepted baseline does not show blocking route continuity issues.
- Schematic-lite is a secondary layout. Phase 4E.1 should make exact shared snapped segments visible with small offsets, but geographic remains the alpha.2 default recommendation.
- Phase 4E.1a trims schematic overlap offsets near station/junction centers; when reviewing schematic-lite, check that nodes do not look twisted by offset route endpoints.
- Phase 4E.1b produced a visual regression even though tests passed; short trimmed fragments and final de-duplication could make routes look broken.
- Phase 4E.1c switches to conservative schematic-lite overlap handling: long safe overlaps may offset, but short or complex junction overlaps fall back to centered connectors. Review continuity first and overlap color visibility second.
- Phase 4E.2 treats the dense `端州火车站` / `小型地铁广场` issue as station spacing. Review whether station markers, route strokes, and labels are more readable after schematic station spacing relaxation.
- Phase 5B starts a topology-first `schematic-v2`. For v2 review, check topology correctness before beauty: stop order, adjacency edges, interchange nodes, and route continuity.
- Phase 5B.3 adds geometry shared corridor diagnostics for schematic-v2. For skip-stop / express services, check whether `pathPoints` reveal a physical corridor even when stop sequences do not fully match.
- Phase 5B.3b materializes geometry shared corridors into schematic-v2 route guides and draws continuous parallel corridor overlays only for materialized shared runs.
- Phase 5B.3c reconstructs the final schematic-v2 render route chain with render-only pass-through guide nodes. For `2号线` / `10号线`, diagnostics/SVG debug attributes should show a materialized route guide and a continuous parallel corridor run; exact node counts can vary with the current conservative corridor guard.
- Phase 5B.4 changes the schematic-v2 strategy for express / rapid / skip-stop variants: draw one canonical route per family, hide variant geometry, and use a white center stripe plus diagnostics to indicate express service. Prefer this stable output over 5B.3 route-chain materialization when reviewing alpha schematic-v2.
- Phase 5B.4 follow-up re-applies corridor route guides to the final schematic-v2 render chain and normalizes obvious canonical backtracking chains. For the primary city, `schematic-v2-service-simplified.svg` should show `10号线` without the previous repeated return path, and `2号线` / `10号线` should appear as a continuous parallel corridor section.
- Phase 5D follow-up guard: exact shared edges alone should not materialize route chains. Geometry corridor materialization should be limited to high-confidence express/service corridors with a real shared adjacent edge, and schematic-v2 parallel overlays should not reappear for unrelated pairs such as `2号线` / `8号线`.
- Phase 5D exact shared platform overlay: final-route-chain segments shared by two-or-more display families may render as `data-schematic-v2-parallel-corridor-source="exact-shared-platform"`. For the primary city, use this to check the west-side `3号线` / `4号线` parallel-track station segment and single-edge shared segments such as `10号线` / `7号线`; it should not add route-guide materialization.
- The same schematic-v2 output should keep the `10号线` white center stripe visible on the shared corridor segment; check for `data-schematic-v2-parallel-corridor-express-marker="true"` if the screenshot is ambiguous.
- Phase 5D.1 freezes the current schematic-v2 candidate under `artifacts\schematic-v2-candidate-freeze\20260604-222351`. Use it as the visual regression reference before accepting future label, badge, legend, or frame polish.
- Phase 5E transit-map readability check: route number badges should avoid station labels and other badges. A badge can be omitted at a crowded endpoint; that is preferable to forcing overlap.
- Phase 5E also adds an `Express service marker` sample to the bottom key when white center stripes are present.
- Schematic-v2 size stability check: Compact, Standard, Poster, Ultra, and Custom sizes should preserve the same schematic-v2 topology and shared-corridor materialization. Size should affect output scale/framing, not whether `2号线` / `10号线` appears as a materialized shared corridor.

## In-Game Export Scenarios

- [ ] No metro city:
  - Load a city with no metro/subway lines.
  - Run `Export Real Metro JSON`.
  - Confirm export does not crash and produces an empty network.
- [ ] Simple 1-line city:
  - Confirm line color, line name, station names, and stop order.
- [ ] Multi-line city:
  - Confirm all metro lines are exported and rendered.
- [ ] Loop line:
  - Confirm loop lines export without crashing and render in both layouts.
- [ ] Branch line:
  - Confirm branch services export and render.
- [ ] Large city with dense center:
  - Confirm Viewer can open the export.
  - Compare `geographic` and `schematic-lite`.
  - In `schematic-lite`, check whether overlapping grid segments show all involved line colors instead of only the last drawn line.
  - Check whether station markers remain clean and centered at overlap junctions.
  - Check whether short overlap fallback preserves route continuity even when it cannot show every color on a tiny connector.
  - Check whether stations that snap too close together are separated enough to read without breaking route topology.
  - Optional: render `schematic-v2` and compare against `schematic-lite`; record whether topology is more trustworthy even if visual polish is still rough.
  - For `schematic-v2`, check whether lines that share an exact station sequence or adjacent station edge stay together until the divergence station.
  - Review `artifacts\schematic-v2-diagnostics\shared-corridors.txt` when a shared corridor looks wrong.
  - Review `artifacts\schematic-v2-diagnostics\geometry-shared-corridors.txt` and `schematic-v2-route-guides.txt` when a shared corridor appears in geographic/pathPoints output but not in stop-sequence diagnostics.
  - Review `artifacts\schematic-v2-diagnostics\schematic-v2-parallel-corridors.txt` to confirm whether a detected shared corridor was materialized and rendered in parallel.
  - For skip-stop / express corridors, verify the follower route polyline actually includes pass-through corridor nodes instead of only showing a short overlay.
  - For Phase 5B.4 review, check `service-family-simplification.txt/csv`; for 10号线 the canonical route should be the station-rich service and the express variants should be listed as hidden variants.
  - For transit-map style review, check that route number badges do not cover station labels or each other, and that the bottom key explains the express white center stripe.
  - Test label filtering options.
- [ ] City with renamed stations:
  - Confirm custom station names appear in the export and Viewer.
- [ ] City with default/generic station names:
  - Confirm `Hide generic station labels` reduces clutter.
- [ ] Airport line / express-service city:
  - Confirm service family merge keeps same-line variants understandable in the legend.
  - Use baseline rendering first.
  - Optionally generate shared corridor / express experimental outputs for comparison.
  - For schematic-v2, check whether express/skip-stop services can follow a physical corridor found from pathPoints instead of drawing as an unrelated independent route.
  - Current primary validation case: `10号线` / `2号线` should appear in geometry shared corridor diagnostics and in `schematic-v2-parallel-corridors` as detected/materialized/parallelRendered. In the latest service-simplified SVG, the 10号线 rendered route should not contain the old backtracking tail.
  - In the same service-simplified SVG, verify that the 10号线 express marker remains visible inside the 2号线 / 10号线 shared section.
- [ ] Dense interchange city:
  - Confirm the baseline SVG/PNG remains usable enough for alpha feedback.
  - Record any route-run fragmentation or visual discontinuity in the feedback notes.

## Error And Edge Cases

- [ ] Invalid JSON:
  - Open a malformed JSON file.
  - Confirm Viewer shows a clear error and does not crash.
- [ ] Missing `metro-export.json`:
  - Temporarily move or rename default export files.
  - Confirm `Open Default Export` is disabled or reports a clear missing-export state.
- [ ] Empty network JSON:
  - Open an empty network sample/manual file.
  - Confirm valid SVG preview appears without crashing.
